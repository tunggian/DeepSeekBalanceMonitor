# Changelog

## 2026-06-26

- 再次重做应用图标为单一 D 形品牌标志加状态孔，原因是上一版余额卡片仍是插画而不是 Logo，缺少可缩小的主记忆点。
- 重绘应用图标为余额卡片和状态硬币组合，原因是上一版抽象科技环语义不清，小尺寸下难以识别这是余额监控工具。
- 新增 Pencil 绘制的应用图标并编译进 EXE，原因是程序需要一个简洁、可识别的静态 Windows 图标，同时保留托盘余额动态图标显示状态。
- 将设置页标题、副标题和行标签改为同一左对齐坐标与无内边距文本绘制，原因是坐标补偿是在猜视觉效果，统一绘制规则才能保证真正左对齐。
- 继续修正设置页标题视觉对齐：标题单独向左补偿到 16px，副标题恢复到行标签轴线，原因是上一版同时移动标题和副标题，截图中标题相对副标题仍然偏右。
- 微调设置页标题视觉左对齐：标题和副标题向左补偿 4px，与下方行标签的视觉边界一致，原因是大号字体左侧字形留白会让相同坐标看起来未对齐。
- 统一设置页行内垂直居中：新增行高居中计算，API Key/刷新间隔输入框、眼睛按钮、开关、主题分段控件和说明文字都按同一行中心对齐，原因是旧布局混用固定 y 偏移导致文字和控件基线不一致。
- 修复设置页打开崩溃和悬浮窗排版：移除设置页控件透明背景用法，避免 WinForms 抛出“不支持透明背景色”；悬浮窗恢复 178px 紧凑宽度并把状态/刷新按钮前移，原因是旧保存宽度和右对齐布局会让余额显示不全且状态区间距过大。
- 修复悬浮窗余额仍被省略和眼睛图标露底的问题：浮窗宽度扩到 210px 并重新分配余额/状态/刷新区域，眼睛按钮绘制前用输入框面板色清底，原因是旧布局主动启用了省略号且图标控件没有参与输入框圆角背景绘制。
- 修复设置页主题预览联动：主题切换会同步预览悬浮窗，取消设置时恢复原主题；同时改用文本渲染器绘制悬浮窗余额并修正圆角输入框背景，原因是上一版只预览设置页且自绘文本/面板仍会裁切或露出矩形底色。
- 修复 DPI 布局回归：开关控件改为按自身高度绘制，说明文字回到控件旁边，悬浮窗余额文字区域占满高度并降低字号，原因是上一版缩放后自绘控件和文字矩形仍按固定像素裁切。
- 修复设置页主题切换交互：点击明亮/黑暗/跟随系统会立即预览当前设置页配色，点击保存才写入配置，点击取消则放弃本次选择，原因是主题选择需要即时反馈但不能绕过确认流程。
- 修复暗色模式下 API Key、刷新间隔输入框和取消按钮 hover 仍使用亮色的问题，并降低悬浮窗余额主文字亮度，原因是深色界面需要一致的可读性和更克制的对比度。
- 增加 WinForms DPI 感知与布局缩放，设置页行标题按控件中心对齐，浮窗尺寸、贴边阈值和内部布局随 DPI 缩放，原因是固定 96 DPI 坐标在高 DPI 屏幕上会错位。

## 2026-06-25

- 修复 API Key 粘贴兼容性：自动清理 `Bearer ` 前缀、引号和首尾空白，减少正确 Key 被拼成错误 Authorization 头导致的 401。
- 新增贴近任务栏的轻量余额状态条，并同步窗口标题和托盘提示，原因是 Win11 任务栏不会稳定展示通知区图标文字。
- 重做设置窗口布局，改用固定表格布局和更宽窗口，避免刷新间隔文字被控件遮挡。
- 调整余额显示为任务栏内嵌式状态块：无边框、不进 Alt-Tab、不显示普通任务栏按钮，右键提供刷新/设置/退出，原因是目标形态应接近网速监控类任务栏文字组件而不是悬浮窗。
- 改为 TrafficMonitor 同类实现：查找 `Shell_TrayWnd` 下的 `ReBarWindow32`/`WorkerW` 和 `MSTaskSwWClass`/`MSTaskListWClass`，将余额窗口挂到任务栏内部并缩短任务按钮区域，退出时恢复，原因是目标是实际嵌入任务栏而不是贴边悬浮。
- 增强任务栏嵌入兼容性：递归查找任务栏子窗口，找不到经典任务按钮区时改为挂到 `Shell_TrayWnd`/可用父窗口并直接定位，同时写入 `%AppData%\\DeepSeekBalanceMonitor\\taskbar.log` 方便定位不同 Win11 版本的任务栏结构。
- 修正任务栏嵌入父窗口：日志显示已找到经典任务栏窗口但未显示，改为挂到 `Shell_TrayWnd` 本体并用 `SetWindowPos(...SWP_SHOWWINDOW)` 强制显示，避免 `ReBarWindow32` 在当前 Win11 上不绘制子窗口。
- 修复任务栏嵌入坐标系错误：余额窗口改回挂到实际父容器并按同一父窗口坐标定位，同时记录父窗口、任务栏、窗口最终坐标，原因是上一版父窗口和定位坐标不一致会导致窗口被放到不可见区域。
- 修正嵌入窗口被父容器裁剪的问题：日志显示余额块超出窄父容器，改为挂到整个 `Shell_TrayWnd` 并按 `TrayNotifyWnd` 左侧定位，避免被 `ReBarWindow32`/任务按钮区裁掉。
- 放弃 Win11 任务栏子窗口嵌入路径，改为无边框顶层覆盖块并定位到系统托盘左侧，原因是日志证明子窗口坐标正确但仍被 Win11 任务栏绘制层吞掉，顶层覆盖才稳定可见。
- 优化任务栏块可见性和样式：改为纯文字两行显示，移除边框和圆形图标，并每秒用 `SetWindowPos(HWND_TOPMOST | SWP_NOACTIVATE)` 维持在托盘左侧，原因是上一版点击其它位置后会被任务栏盖住且视觉像小卡片。
- 修正上一版样式和保活逻辑未生效的问题：重写任务栏状态窗，实际启用 500ms 置顶保活、无激活显示、纯文字居中两行和紧凑尺寸，原因是机械替换没有改到运行代码。
- 移除不稳定的 WinForms 任务栏覆盖块，恢复为稳定托盘图标显示，并新增原生任务栏集成说明文档，原因是 Win11 任务栏文字块需要 Explorer/DeskBand 级原生组件，当前轻量 EXE 路线不可维护。
- 改为普通桌面悬浮窗：新增 Win11 云母/圆角属性、桌面浮窗余额卡片、拖拽吸附屏幕边缘和贴边隐藏，原因是任务栏嵌入路线不稳定且用户改为悬浮窗形态。

## 2026-06-24

- 新增 Windows 11 风格的 DeepSeek API 余额托盘监控程序：可在任务栏通知区域显示余额状态，支持双击/菜单打开设置窗口。
- 新增 DeepSeek `/user/balance` 余额查询、API Key 本机用户加密保存、刷新间隔设置，默认 1 分钟刷新一次。
- 新增本地构建脚本和核心逻辑自检，原因是最终交付目标是轻量 EXE，需要可重复生成和验证。
